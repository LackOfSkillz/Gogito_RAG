Purple Wi-Fi and Wi-Fi Cloud Integration Guide

This guide demonstrates how to integrate WatchGuard Wi-Fi Cloud with Purple Wi-Fi authentication to enable guest users to authenticate to a captive portal.

Purple Wi-Fi Configuration

  1. Log in to the Purple Wi-Fi Management Portal.
  2. Click .

  1. Select Management > Locations (or Venues, depending on the browser language settings).

  1. Select Venues and Groups.
  2. Click > Add venue.

  1. Follow the instructions in the wizard to add a new venue.
  2. Select your new venue from Venues and Groups.
  3. From your venue, select Hardware.
  4. Click Add hardware > Add new hardware.
  5. From the WiFi AP type drop-down list, select WatchGuard Wi-Fi Cloud AP.
  6. In the Name text box, edit the name of the AP if required. The default name is added automatically when you select the Hardware type.
  7. In the WiFi AP Mac Address text box, add the MAC address of your WatchGuard AP.
  8. Click View Manual Online or Download Manual to view the configuration steps and information required for the WatchGuard Wi-Fi Cloud configuration.

  1. Click Save details.

Configure WatchGuard Wi-Fi Cloud to Integrate with Purple Wi-Fi

The information required in these steps is detailed in the manual that you downloaded from Purple Wi-Fi in the previous step. For more information, go to Purple Wi-Fi Configuration.

Add RADIUS Servers

To add RADIUS servers to Wi-Fi Cloud:

  1. Log in to WatchGuard Wi-Fi Cloud.
  2. Open Discover.
  3. Select the location where to apply the settings.
  4. Select Configure > Network Profiles > RADIUS.
  5. Click Add RADIUS Server.
  6. Type the RADIUS Server Name, IP Address/FQDN, Authentication Port, Accounting Port, and Shared Secret of the Purple RADIUS server.

  1. Click Save.
  2. Repeat these steps for the second Purple Wi-Fi RADIUS server.

Add an SSID

To add an SSID in Wi-Fi Cloud:

  1. Open Discover.
  2. Select the location where to apply the settings.
  3. Select Configure > WiFi, then select the SSID tab.
  4. Click Add SSID.
  5. Type an SSID Name and Profile Name.
  6. In the Select SSID Type section, select Guest.

  1. Select the Security tab.
  2. From the Select Security Level for Associations drop-down list, select Open.

  1. Select the Network tab.
  2. Select NAT, then configure the IP address range and DNS server details:
  • Start IP Address
  • End IP Address
  • Local IP Address
  • Subnet Mask
  • Lease Time
  • DNS Servers


  1. Select the Captive Portal tab.
  2. Enable Captive Portal.
  3. From the Cloud Hosted drop-down list, select Third-Party Hosted.
  4. Select With RADIUS Authentication.
  5. In the Splash Page URL text box, type or paste the URL provided by Purple Wi-Fi.
    For example: https://region1.purpleportal.net/access/

  7. Clear the HTTPS Redirection check box.

    For example: https://region1.purpleportal.net/access/?res=success
  2. Click the RADIUS Settings link.
  3. From the Primary and Additional Authentication Server and Accounting Server drop-down list, select the RADIUS server configuration you created for Purple Wi-Fi.
  4. Set the Accounting Interval to 2 minutes.
  5. Set the Called Station ID to %m.

The SSID will be applied to APs in the selected location. Make sure you configure radio and device settings for your APs if you have not already configured these settings. For more information, go to Configure Radio Settings.

Test Purple Wi-Fi Integration

To test the Purple Wi-Fi integration with Wi-Fi Cloud:

  1. From your wireless client device, to connect to the SSID you created in Wi-Fi Cloud.
  2. Your browser redirects to the Purple Wi-Fi splash page.
  3. Log in with your existing Purple Wi-Fi account or you can register for a new account.
  4. After successful authentication, you can connect to the Internet through the captive portal.
    For the wireless client details, go to Marketing > CRM and filter the CRM records.
