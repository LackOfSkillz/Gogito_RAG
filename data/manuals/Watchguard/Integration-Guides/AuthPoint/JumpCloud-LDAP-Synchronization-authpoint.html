JumpCloud LDAP Synchronization with AuthPoint

Deployment Overview

This document describes how to sync LDAP users from JumpCloud to AuthPoint. To sync users from JumpCloud, you must add an LDAP external identity in AuthPoint and create one or more queries. You must also download and install the AuthPoint Gateway.

In AuthPoint, LDAP external identities represent external user databases. They connect to user databases to get user account information and validate passwords. The queries you add to an external identity specify which users to sync from. They pull user information and create AuthPoint user accounts for the users that are found.

Configure JumpCloud

  1. Log in to the JumpCloud interface as an administrator.
  2. Select User Management > Users.
  3. Click to add or import users. In our example, we manually add an admin user. This user account is used by AuthPoint to find and sync other users.
  4. In the User Information section, you must complete these fields:
    • First Name — Type a first name for this user account.
    • Last Name — Type a last name for this user account.
    • Username — Type a user name for this user account.
    • Company Email — Type a valid email address for this user account.

  1. In the User Security Settings and Permissions section, select the Specify initial password check box and type a password in the Password and Confirm Password text boxes. You will use this password for this user when you configure an external identity in AuthPoint.
  2. Select the Enable as LDAP Bind DN check box.

  1. Click Save user.
    The user account is created.
  2. Go to Details page for this user and copy the LDAP Distinguished Name value. You will use this value in the next section when you configure an external identity in AuthPoint.

In our example, we create two additional user accounts.

  1. Select User Management > User Groups.
  2. Click to add a group.
  3. Type a name for this group. In our example, we name this group EcoTest.

    The name of this group must match the name of your group in AuthPoint.

  1. Select the Users tab.
  2. Select the users that you want to sync to AuthPoint. In our example, we select the admin user and the other two users that we created.

    If you want to add your users to different groups in AuthPoint, you must create multiple groups for your users in JumpCloud. You might do this if you want to different groups to have access to different resources or if you want to have different MFA options for different groups.

  1. Click Save.

  1. Select User Authentication > LDAP.
  2. Select JumpCloud LDAP,
  3. From the Details tab, copy the ORG DN value. You use this value when you add an advanced query in AuthPoint to sync your users.

  1. Select the User Groups tab.
  2. Select the user groups that you will sync to AuthPoint. In our example, we select the EcoTest group that we created.

  1. Select the Users tab.
  2. Select and bind your users.
  3. Click Save.

Configure AuthPoint

To configure AuthPoint to sync users from JumpCloud, you must:

  • Add an LDAP external identity
  • Download and install the AuthPoint Gateway
  • Create one or more advanced queries

Add an External Identity

  1. In a web browser, go to cloud.watchguard.com and log in to your WatchGuard Cloud account.
  2. Select Configure > AuthPoint. If you have a Service Provider account, you must select an account from Account Manager.
  3. Select External Identities.
  4. From the Choose an External Identity Type drop-down list, select LDAP. Click Add.

  1. In the Name text box, type a descriptive name for the external identity.
  2. In the LDAP Search Base text box, type your LDAP root domain name. This is based on the LDAP Distinguished Name value from JumpCloud that you copied in the previous section. You must remove "uid=UserName" from the LDAP Distinguished Name value.

  1. Enable the toggle above the System Account text box.
  2. In the System Account DN text box, type the full LDAP Distinguished Name value that you copied from JumpCloud in the previous section. In our example, this value is uid=Jumpcloudadmin,ou=Users,o=xxxxxxx,dc=jumpcloud,dc=com.

    This user must have permissions to perform LDAP searches and binds.

  3. In the Passphrase text box, type the password for this user. In our example, this is the password we defined in Step 5 of the previous section.
  4. From the Synchronization Interval drop-down list, specify how often you want to synchronize the users from the LDAP database. If you select Every 24 hours, you must also specify what time the synchronization starts each day.

  1. For Type, select Others.

  1. In the Domain text box, type jumpcloud.com.
  2. Type a value for each attribute.
    1. In the Attribute related to the first name text box, type givenName.
    2. In the Attribute related to the last name text box, type sn.
    3. In the Attribute related to the user email text box, type mail.
    4. In the Main attribute to the LDAP user text box, type uidNumber.
    5. In the Attribute related to the user login text box, type uid.
    6. In the Attribute related to the mobile number text box, type homePhone.

    You might need to change some of these values based on how your organization has configured JumpCloud.

  1. In the Server Address text box, type ldap.jumpcloud.com.
  2. Make sure the LDAPS toggle is enabled.
  3. In the Server Port text box, type 636.

  1. Click Save.

Configure and Install the AuthPoint Gateway

The LDAP external identity must be added to the configuration for a Gateway that is installed on your corporate network and has access to the JumpCloud server.

If you do not have an existing Gateway, you must add one.

  1. From the navigation menu, select Gateway.
  2. Click the Name of your Gateway. If you do not have an existing Gateway, click Add Gateway.
  3. If this is a new Gateway, type a name for the Gateway.
  4. In the LDAP section, from the Select an LDAP external identity list, select the external identity you created in the previous section.

  1. Click Save.
  2. If this is a new Gateway, you must download and install the AuthPoint Gateway on your network in a location that has Internet access and can connect to your JumpCloud server. For detailed steps to install a Gateway, see to the Download and Install the Gateway section in the About Gateways help topic.

    When you install the AuthPoint Gateway, you must have the registration key for your Gateway. To see this value, click the Registration Key link at the bottom of the tile for your Gateway on the Gateway page in AuthPoint.

Create an Advanced Query and Sync Your Users

After you create an external identity for JumpCloud, you must add an advanced query to specify:

  • The JumpCloud users to sync
  • The AuthPoint group to add the users to

After you add an advanced query to find your users, AuthPoint syncs with JumpCloud at the next synchronization interval This is defined in the Synchronization Interval drop-down list on the configuration page for your external identity. and creates an AuthPoint user account for each user identified by the query. If the query returns more users than you have available licenses for, AuthPoint only creates as many users as your license supports.

LDAP users that do not have a name, user name, or email address are not included in the synchronization.

Make sure that each user account has a valid email address. If the email address for a user account is not correct, the user cannot receive the email message to activate a token.

To add an advanced query:

  1. From the navigation menu, select External Identities.
  2. Next to the external identity you added for JumpCloud, click and select Check Connection. This indicates whether AuthPoint can connect to JumpCloud.
  3. Next to the external identity you added for JumpCloud, click and select Advanced Query.
  4. Click the Add Advanced Query.
  5. In the Name text box, type a descriptive name for the query.
  6. From the Group drop-down list, select the AuthPoint Group to add the users to. This AuthPoint group name must match the name of the JumpCloud group that you sync users from.
  7. In the Advanced Query text box, type memberOf=cn=<Your Group Name>,ou=Users,<Your ORG DN>. We copied the ORG DN value from JumpCloud in the Configure JumpCloud section.

  1. Click Validate Advanced Query to preview your query results. You can see the number of users your query returns and a preview of the first 10 users.

    No users are synced when you validate a query. Users are only synced after you have added your query to the external identity and saved the changes.

  1. Click OK.
  2. Click Add.

  1. (Optional) To sync your users now, on the External Identities page, next to the your external identity, click and select Start Synchronization. If you do not sync your users now, AuthPoint syncs with JumpCloud at the next synchronization interval This is defined in the Synchronization Interval drop-down list on the configuration page for your external identity..

An AuthPoint user account is created for each user identified by the advanced query. The created user accounts appear on the Users page. You can identify users synced from an external identity by the LDAP tag next to their name in the list of users.

Each user is sent an email that they use to activate their token in the AuthPoint mobile app. When a user activates their token, their token information is shown in the Token column with a green Activated status icon next to the token.
